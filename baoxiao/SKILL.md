# 报销处理

用于处理一组个人报销材料，源材料通常包括：

- 每人一份 `pdf` 发票
- 每人一张或多张 `jpg/png` 支付截图
- 一个目标 `xlsx` 模板

优先保留模板样式，只修改单元格内容和必要的表头。

## 触发场景

当用户要求以下任一事项时使用本技能：

- 将多人报销汇总到一个 Excel 表
- 按指定汇率生成 `HKD` 或 `CNY` 报销版本
- 保留 `Invoice` 原始币别和金额，同时重算报销金额
- 另存为指定命名规则的新文件

## 工作流

1. 找到材料

- 枚举工作区内所有 `pdf/jpg/jpeg/png`
- 以文件名前缀归类到具体人员；如 `张三.pdf`、`张三.jpg`、`张三_1.png`
- 确认目标表格路径和现有表头

2. 读取模板

- 优先用 `openpyxl` 读取 `xlsx`
- 若环境缺少 `openpyxl`，使用 `python3 -m pip install --target /tmp/openpyxl_pkg openpyxl`
- 读取首行表头、样式、总额公式位置
- 不手改 `xlsx` 的 XML，优先通过 `openpyxl` 回写

3. 提取账单信息

- PDF 发票：优先转图片后 OCR
- 图片截图：直接 OCR
- 可用命令：
  - `sips -s format png input.pdf --out /private/tmp/out.png`
  - `tesseract image stdout -l chi_sim+eng --psm 6`
- 必提字段：
  - 使用人
  - 费用发生月
  - 实际支付币别
  - 实际支付金额
  - Invoice 币别
  - Invoice 金额

4. OCR 纠错

- OCR 结果不可靠时，直接查看原图核对关键金额
- 重点核对：
  - 币种是否被误识别
  - `20.00`、`19.00`、`137.61` 这类金额
  - `USD`、`SGD`、`CNY` 等币别
- 发票币别与支付币别不一致时，两者都要保留

5. 回填 Excel

- 明细从第 2 行开始
- 如明细行数不足，插入新行并复制模板样式
- 典型列映射：
  - A: 序号
  - B: 费用发生月
  - C: 使用人
  - D: 实际支付币别
  - E: 实际支付金额
  - F: Invoice 币别
  - G: Invoice 金额
  - H: 中行折算价
  - I: 报销金额
- 总额行保留在明细底部，公式更新为 `=SUM(I2:I{最后一行})`

## 汇率规则

按用户明确给出的汇率计算，不自行猜测或联网替换。

示例：

- `HKD:CNY = 100:88.3`，则 `1 HKD = 0.883 CNY`
- `USD:CNY = 100:691.94`，则 `1 USD = 6.9194 CNY`
- `USD:HKD = 6.9194 / 0.883 = 7.8362400906`

金额保留两位小数，使用常规四舍五入。

## 两种常见输出口径

### HKD 报销版

适用于用户要求“报销 HKD”：

- `Invoice币别`、`Invoice金额` 必须如实填写，不要改成 `HKD`
- `I1` 改为 `报销金额-HKD`
- `报销金额-HKD` 按 `实际支付金额` 换算
- `中行折算价` 根据实际支付币别填写：
  - 实付为 `USD`：填 `USD:HKD`
  - 实付为 `CNY`：填 `CNY:HKD`，即 `1 / 0.883`

公式：

- 实付 `USD`：`实际支付金额 * USD_TO_HKD`
- 实付 `CNY`：`实际支付金额 / HKD_TO_CNY`

### CNY 报销版

适用于用户要求“报销 CNY”：

- `I1` 改为 `报销金额-CNY`
- 默认按 `Invoice金额` 和 `Invoice币别` 换算为 `CNY`
- 若 `Invoice币别` 为 `SGD` 且用户要求特殊处理，则改按 `实际支付金额` 和 `实际支付币别` 计算

本次流程中的特殊规则：

- 正常 `USD` 发票：`报销金额-CNY = Invoice金额 * USD_TO_CNY`
- `SGD` 发票：按实际支付 `USD` 金额计算，即 `实际支付金额 * USD_TO_CNY`

## 文件命名

按用户指令生成新文件，不覆盖原文件，除非用户明确要求覆盖。

本次已验证的命名模式：

- `报销{当前月份}月cursor_实际支付.xlsx`

其中“当前月份”取当前系统日期的月份数字，例如 `2026-04-13` 对应 `报销4月cursor_实际支付.xlsx`

## 回写和校验

回写前先生成预览副本，例如：

- `报销-HKD预览.xlsx`
- `报销-HKD修正预览.xlsx`
- `报销4月cursor_实际支付.xlsx`

校验项：

- 表头是否符合用户要求
- `Invoice` 币别和金额是否保留原值
- `中行折算价` 是否与“实际支付币别 -> 报销币别”一致
- 最后一列格式是否正确
- 总额公式是否覆盖所有明细行
- 总额是否与逐行求和一致

## 注意事项

- 不要因为“报销 HKD”就把 `Invoice` 币别和金额改成 `HKD`
- `中行折算价` 的方向要和报销币种一致，不能混填
- `SGD` 发票若无用户提供的 `SGD:CNY` 汇率，不要自行假设；按用户指定规则处理
- 编辑桌面文件前，先在工作区生成预览文件，再复制到目标路径
- 若需要覆盖桌面文件，先确认覆盖的是哪一个文件
