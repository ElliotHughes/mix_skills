# 九、工时评估（强制输出）

## 评估口径（默认）

- 估算对象：后端主程（熟悉该仓库、熟悉 Yii2 模块化、能独立完成联调）
- 单位：小时（h），同时给出人日（d，按 1d=7.5h）
- 输出必须按“接口粒度”逐条评估（每个 action / endpoint 一行）
- 每项给出：乐观 / 常规 / 保守 三档，用于应对不确定性
- 估算必须包含：开发 + 自测 + Code Review 修改成本
- 如果存在 DB 变更、权限变更、跨模块联动、外部依赖（支付/客户端/RPA），必须上调保守值并注明原因

## 评估维度与加权规则

对每个接口改动，按以下维度打勾并累计复杂度点数（Complexity Points）：

### A. 改动类型基础分（必选其一）
- 新增接口（新增 Controller action + 完整链路）：3 分
- 修改接口参数/逻辑（保持路由不变）：2 分
- 仅修复 bug（逻辑小改，不改契约）：1 分
- 删除接口 / 不兼容变更（含迁移方案）：4 分

### B. 输入/校验复杂度（可叠加）
- 新增/修改字段 1-3 个（含 validator）：+1
- 新增/修改字段 4-10 个：+2
- 字段 >10 或校验规则复杂（条件校验/数组嵌套/跨字段校验）：+3

### C. 输出契约变更（可叠加）
- 响应 data 结构新增字段（兼容）：+1
- 响应 data 结构改变/字段语义改变（潜在破坏）：+2
- code/msg 语义调整或错误码体系变更：+2

### D. 权限/鉴权/风控（可叠加）
- 需要鉴权但逻辑不变（AuthBase/ClientBase）：+1
- 权限模型变化/新增角色校验/资源归属校验：+2

### E. 数据库与数据处理（可叠加）
- 仅查询（db_read）无新增索引：+1
- 写入/事务/并发一致性处理：+2
- 表结构变更（migration + 回填/兼容读写）：+4
- 影响多表关系/复杂联表/索引优化：+3

### F. 外部系统/跨模块联动（可叠加）
- 跨模块调用（common/service 复用等）：+1
- 依赖支付/客户端协议/RPA 执行链路等外部系统：+3

### G. 风险与不确定性（可叠加）
- 需求不明确/需要反复确认：+2
- 线上灰度/兼容旧版本/回滚策略复杂：+2

## Complexity Points → 工时换算

将总分 CP 映射为工时（后端主程节奏）：

- CP 1-2：乐观 1h / 常规 2h / 保守 4h
- CP 3-4：乐观 2h / 常规 4h / 保守 8h
- CP 5-6：乐观 4h / 常规 8h / 保守 12h
- CP 7-8：乐观 8h / 常规 12h / 保守 16h
- CP 9-10：乐观 12h / 常规 16h / 保守 24h
- CP 11+：乐观 16h / 常规 24h / 保守 40h（建议拆分任务/分阶段发布）

> 注：如果同时包含“表结构变更 + 不兼容变更”，保守值至少 24h 起。

## 输出表格格式（必须）

在“接口变更说明”后，新增“工时评估表”，字段固定如下：

| 模块 | 控制器 | 方法 | 变更点摘要 | CP | 乐观(h) | 常规(h) | 保守(h) | 关键不确定性/风险 |
|------|--------|------|------------|----|---------|---------|---------|-------------------|

并在表格后输出：

- 总工时（常规）= 所有接口常规值求和
- 建议排期（人日）= 总常规工时 / 8（向上取整）
- 若存在 CP 11+ 的接口：必须给“拆分建议/分阶段发布建议”
